Syntax Guide (V. 0.1)

Syntax for OpenBooks is defined by hashtags, hashtags were chosen because they are unary symbol thats easy to see. Tokens can be written inside the pair of hashtags examples are par, img, link... Options are delimted by a pipe symbol '|'. There are two types of options, commands and parameters. Commands are single words that can apply to multiple different tags. Parameters are more specific and require what you are specifying along with the specific action. Here are some examples :

centered par 
	#par | center# content here ...
image with alt text and a src 
	#img | src = URL (can be a link or file location)
	 alt= alt text (used for screen-readers) #

Since the hashtag is reserved, you can escape it with a backslash to print it without entering a new tag e.g '\#' results in \#

Paragraph Tags 

	#par# content here...

Accepts the following options

  • Left, Center, Right

Images 

	#img | src = src_url#

images display pictures and gifs. They can either be local files (internal) or links (external). By default images are searched in the media folder generated by OpenBooks.

Note: the extension is optional, OpenBooks will target matching file basenames if an extension is not specified

To look outside of the media folder, you can enter an absolute path

Note: web versions of the compiler are expected to restrict this for security reasons Accepts the following options
  • Left, Center, Right
  • src = src_url (this is required)
  • alt = alt_text used for screen readers, by default the alt text will be extracted from the file name
